# ADR-006: Error response code field is numeric HTTP status

## Context

Q-013 asks whether the error code field should be numeric (401) or string ("UNAUTHORIZED").
The PRD example shows "<HTTP_STATUS>" which is ambiguous.


## Decision

Use numeric HTTP status code as the code field value. Example: { error: { code: 401, message: "...",
requestId: "..." } }. This matches the HTTP status code in the response line, is unambiguous, and
follows Google Cloud API conventions.


## Consequences

+ Consistent with GCP API error format that callers are likely familiar with.
+ No mapping table needed — code mirrors the HTTP status.
+ Simplifies client error handling (switch on numeric code).
