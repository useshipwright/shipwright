# ADR-006: Correlation ID: propagate or generate

## Context

REQ-014 requires correlation ID on all requests. Q-007 asks whether to generate or propagate from incoming headers.


## Decision

Propagate X-Request-ID from incoming request header if present. Generate UUID v4 if absent. Return X-Request-ID in response headers always. Attach to request object as request.correlationId for use in logging and response construction.


## Consequences

- Supports distributed tracing when callers propagate their request IDs
- Self-sufficient — generates IDs when called without one
- X-Request-ID is the de facto standard header (not X-Correlation-ID)
