# ADR-012: ESM modules with TypeScript

## Context

Node 20 supports ESM natively. Fastify and Firebase Admin SDK support ESM imports.


## Decision

Use ESM throughout. tsconfig.json targets ES2022 with module NodeNext and moduleResolution NodeNext. package.json sets "type": "module". All imports use .js extension in source (TypeScript resolves to .ts during compilation). This aligns with modern Node.js conventions and avoids CJS/ESM interop issues.


## Consequences

- Clean ESM — no require() except in test helpers (createRequire for resolving template deps)
- Import extensions (.js) required in source files
- Vitest handles ESM natively
