# ADR-014: PRD Correction: Cloud Build Service Account Roles

## Context

PRD validation found a gap: the PRD does not list Cloud Build service account roles. GCP docs specify Cloud Build SA needs roles/artifactregistry.writer (push images), roles/run.developer (deploy to Cloud Run), and roles/iam.serviceAccountUser (act as Cloud Run SA).

**Source:** gcp — overview.md § Cloud Build Service Account

## Decision

Add Cloud Build SA roles to the IAM specification: roles/artifactregistry.writer, roles/run.developer, roles/iam.serviceAccountUser. These are defined in iam/custom-roles.yaml as the firebaseAuthPackBuilder custom role.

## Consequences

- PRD section '9) IAM Requirements' is supplemented with Cloud Build SA roles
- Implementation follows verified GCP documentation
- Cloud Build can push images and deploy to Cloud Run without manual IAM fixes
