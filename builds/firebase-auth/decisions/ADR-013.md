# ADR-013: PRD Correction: Operator IAM Requirements

## Context

PRD validation found a mismatch between the PRD and domain documentation.

**PRD claims:** Operator needs roles/run.admin, roles/cloudbuild.builds.editor, roles/secretmanager.admin, roles/iam.serviceAccountAdmin, roles/resourcemanager.projectIamAdmin

**Documentation says:** GCP docs list two additional required roles: roles/artifactregistry.admin (create Artifact Registry repositories, push container images) and roles/iam.serviceAccountUser (iam.serviceAccounts.actAs permission for deploying Cloud Run services). Without these, the build/deploy pipeline fails.

**Source:** gcp — overview.md § Permissions & Scopes — Operator

## Decision

Add roles/artifactregistry.admin and roles/iam.serviceAccountUser to the operator role set. Keep roles/resourcemanager.projectIamAdmin (reasonable for binding IAM roles at the project level, even though not in GCP docs).

Full operator roles: roles/run.admin, roles/cloudbuild.builds.editor, roles/secretmanager.admin, roles/artifactregistry.admin, roles/iam.serviceAccountUser, roles/iam.serviceAccountAdmin, roles/resourcemanager.projectIamAdmin.

## Consequences

- PRD section '9) IAM Requirements — Operator' is overridden
- Implementation follows verified GCP documentation
- Deploy pipeline succeeds without manual role discovery
