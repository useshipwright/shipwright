# ADR-007: Log redaction format

## Context

REQ-007 requires partial redaction of emails and UIDs. Q-008 asks about the specific format.


## Decision

Emails: first character + *** + @domain (e.g., t***@example.com). UIDs: first 4 characters + *** (e.g., abc1***). Tokens: first 10 characters + [REDACTED] (enough to identify token type, not enough to reconstruct). Service account JSON: [REDACTED_CREDENTIAL]. Redaction is applied via Pino serializers, not string replacement in application code.


## Consequences

- Logs retain enough info for debugging (which user, which token format) without exposing full PII
- Pino serializer approach means redaction is automatic — developers cannot accidentally log unredacted data
- Performance cost is minimal — serializers run only when log entry is written
