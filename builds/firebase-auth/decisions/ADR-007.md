# ADR-007: Firebase SDK errors mapped to 502 for infrastructure failures

## Context

Q-014 asks how to handle Firebase SDK errors beyond user-not-found (network errors, quota exceeded).
These are infrastructure failures, not client errors.


## Decision

Map Firebase error codes to HTTP status codes: auth/user-not-found → 404, auth/quota-exceeded → 503,
auth/internal-error → 502, auth/insufficient-permission → 500, network errors → 502. The error-handler
plugin logs the full Firebase error server-side and returns a generic message to the client. For 502/503,
include Retry-After header when estimable.


## Consequences

+ Callers can distinguish between client errors (4xx) and infrastructure problems (5xx).
+ 502 signals "upstream dependency failed" which is semantically correct for an API proxy.
+ Generic messages on all 5xx prevent information leakage about Firebase infrastructure.
