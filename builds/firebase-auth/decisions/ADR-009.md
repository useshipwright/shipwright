# ADR-009: PRD Correction: §6 CAP-4: Custom Claims

## Context

PRD validation found a mismatch between the PRD and domain documentation.

**PRD claims:** Maximum 1000 bytes serialized

**Documentation says:** INCORRECT. The limit is 1000 characters based on JSON.stringify().length (JavaScript string length / UTF-16 code units), NOT UTF-8 byte length. Source code checks: request.customAttributes.length > MAX_CLAIMS_PAYLOAD_SIZE where MAX_CLAIMS_PAYLOAD_SIZE = 1000. Multi-byte Unicode characters count as 1 character each (unless surrogate pairs).

**Source:** firebase-auth — overview.md

## Decision

Follow the domain documentation behavior, not the PRD claim. The implementation should: INCORRECT. The limit is 1000 characters based on JSON.stringify().length (JavaScript string length / UTF-16 code units), NOT UTF-8 byte length. Source code checks: request.customAttributes.length > MAX_CLAIMS_PAYLOAD_SIZE where MAX_CLAIMS_PAYLOAD_SIZE = 1000. Multi-byte Unicode characters count as 1 character each (unless surrogate pairs).

## Consequences

- PRD section '§6 CAP-4: Custom Claims' is overridden
- Implementation follows verified API behavior
- Reduces risk of runtime errors from incorrect assumptions
