# ADR-009: Smoke tests authenticate via gcloud run services proxy

## Context

Q-009 asks how smoke tests authenticate to an internal-only Cloud Run service. The service has --no-allow-unauthenticated and --ingress=internal.


## Decision

The deploy script uses `gcloud run services proxy` to create a localhost tunnel to the internal service. Smoke tests curl localhost. The proxy authenticates using the operator's gcloud credentials. For CI, the operator service account has roles/run.invoker on the deployed service. Alternative: temporarily set allow-unauthenticated during smoke tests — rejected due to security risk window.


## Consequences

- No security window — service is never publicly accessible
- Requires gcloud SDK on the machine running smoke tests
- Proxy adds slight latency but smoke tests are not performance-critical
- CI runners need authenticated gcloud with run.invoker role
