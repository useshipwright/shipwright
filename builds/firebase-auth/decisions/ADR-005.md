# ADR-005: Pino + @google-cloud/pino-logging-gcp-config for structured logging

## Context

REQ-013 requires structured JSON logs compatible with Cloud Logging. Fastify uses Pino natively. GCP research identifies @google-cloud/pino-logging-gcp-config as the recommended integration.


## Decision

Use Fastify's built-in Pino logger with @google-cloud/pino-logging-gcp-config for severity mapping (info→INFO, error→ERROR), message key ('message' not 'msg'), and timestamp format. Add custom Pino serializers from lib/redact.ts for email/UID/secret redaction. Inject correlationId into child logger per request via onRequest hook.


## Consequences

- Logs are structured JSON, auto-ingested by Cloud Logging with correct severity
- Redaction applies to all log levels via Pino serializers — cannot be bypassed by route code
- correlationId in every log entry enables cross-service request tracing
