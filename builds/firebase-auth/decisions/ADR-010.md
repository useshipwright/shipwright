# ADR-010: PRD Correction: §10 Security Principles

## Context

PRD validation found a mismatch between the PRD and domain documentation.

**PRD claims:** Custom claims payload size (max 1000 bytes)

**Documentation says:** INCORRECT. Same issue as §6 CAP-4: the limit is 1000 characters (JSON.stringify length), not 1000 bytes.

**Source:** firebase-auth — overview.md

## Decision

Follow the domain documentation behavior, not the PRD claim. The implementation should: INCORRECT. Same issue as §6 CAP-4: the limit is 1000 characters (JSON.stringify length), not 1000 bytes.

## Consequences

- PRD section '§10 Security Principles' is overridden
- Implementation follows verified API behavior
- Reduces risk of runtime errors from incorrect assumptions
