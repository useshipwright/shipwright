# ADR-001: Adapter pattern for Firebase Admin SDK

## Context

Route handlers need to call verifyIdToken() and getUser(), but these require real Firebase credentials and network access. Tests must run without Firebase. The Firebase credential must be isolated per REQ-006.


## Decision

Wrap Firebase Admin SDK in adapters/firebase-admin.ts. This module is the ONLY place that imports firebase-admin and handles credential init. The firebase plugin reads the credential from env, calls the adapter, and decorates Fastify with the Auth instance. Route handlers access Firebase only through the app.firebaseAuth decorator. Tests mock the adapter module.


## Consequences

- Secret isolation (REQ-006) enforced architecturally — grep for firebase-admin imports
- Tests can mock a single module instead of the entire Firebase SDK
- Slight indirection cost — acceptable for a 4-route service
