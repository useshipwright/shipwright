# ADR-001: Adapter pattern for Firebase Admin SDK

## Context

The Firebase Admin SDK is a complex dependency with network calls, caching, and internal state.
Direct usage in route handlers would make testing require either the Firebase emulator or 
complex module mocking (which breaks with ESM imports and firebase-admin's internal structure).


## Decision

Wrap all Firebase Admin Auth methods in a single firebase-adapter module (src/infra/firebase-adapter.ts).
This is the ONLY module that imports firebase-admin. All route handlers and use cases depend on the
adapter interface, not the SDK directly. Tests mock the adapter, not the SDK.


## Consequences

+ Tests are fast and reliable — no Firebase emulator needed for unit/integration tests.
+ Secret isolation enforced architecturally — only config + adapter touch the credential.
+ Adapter methods can be instrumented with metrics transparently.
- One extra layer of indirection (minimal overhead — methods are thin async wrappers).
- Adapter must be kept in sync with SDK API surface (9 capabilities, ~18 methods — manageable).
