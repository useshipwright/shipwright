# ADR-008: Batch user lookup request schema uses discriminated union identifiers

## Context

Q-007 flags that POST /users/batch request body is not defined. Firebase SDK's getUsers() accepts
an array of UserIdentifier union type objects ({uid}, {email}, or {phoneNumber}).


## Decision

Mirror the Firebase SDK's identifier format exactly:
{ identifiers: [{ uid: "..." }, { email: "..." }, { phoneNumber: "+1..." }] }
Each identifier object must have exactly one key (uid, email, or phoneNumber). Validated with
JSON Schema oneOf. This avoids inventing a new format and maps 1:1 to the SDK call.


## Consequences

+ Zero transformation needed between API input and Firebase SDK call.
+ Callers familiar with Firebase SDK will recognize the format immediately.
+ JSON Schema validation catches malformed identifiers before hitting the SDK.
