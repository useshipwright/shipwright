# ADR-008: Multi-stage Dockerfile with node:20-slim

## Context

REQ-024 requires containerised build. GCP research recommends multi-stage builds with node:20-slim or distroless.


## Decision

Three-stage Dockerfile: (1) deps — install pnpm via corepack, copy lockfile, pnpm fetch + install --frozen-lockfile; (2) build — copy source, run tsc; (3) runtime — node:20-slim, copy built JS + node_modules, run as non-root user. Use .dockerignore to exclude tests, .git, docs. Use pnpm deploy --prod for minimal node_modules in runtime stage.


## Consequences

- Small runtime image (~150MB vs ~900MB for full node:20)
- Build reproducibility via --frozen-lockfile
- Non-root container user for security
- pnpm deploy --prod eliminates devDependencies from runtime
