# ADR-011: PRD Correction: §11 IAM Requirements

## Context

PRD validation found a mismatch between the PRD and domain documentation.

**PRD claims:** Pack service account: roles/secretmanager.secretAccessor, roles/logging.logWriter, roles/firebaseauth.admin

**Documentation says:** roles/firebaseauth.admin is confirmed as 'Full read/write access'. However, the docs note TWO additional requirements not listed in the PRD: (1) createCustomToken() requires iam.serviceAccounts.signBlob permission (via roles/iam.serviceAccountTokenCreator), and (2) firebase-admin v12+ requires serviceusage.services.use permission (via roles/serviceusage.serviceUsageConsumer).

**Source:** firebase-auth — overview.md

## Decision

Follow the domain documentation behavior, not the PRD claim. The implementation should: roles/firebaseauth.admin is confirmed as 'Full read/write access'. However, the docs note TWO additional requirements not listed in the PRD: (1) createCustomToken() requires iam.serviceAccounts.signBlob permission (via roles/iam.serviceAccountTokenCreator), and (2) firebase-admin v12+ requires serviceusage.services.use permission (via roles/serviceusage.serviceUsageConsumer).

## Consequences

- PRD section '§11 IAM Requirements' is overridden
- Implementation follows verified API behavior
- Reduces risk of runtime errors from incorrect assumptions
