# ADR-003: Cloud Scheduler for calendar sync cron instead of in-process timer

## Context

Q-010/Q-016 identified that Cloud Run is request-driven and doesn't support native cron. Options: Cloud Scheduler hitting an internal endpoint, setInterval in the process (requires cpu-always-allocated), or a separate Cloud Function.


## Decision

Use Cloud Scheduler to POST to /internal/calendar-sync every 5 minutes. Authenticate with OIDC token targeting the Cloud Run service account. The endpoint iterates all users with calendarConnected=true and performs incremental sync.


## Consequences

Pro: Reliable, observable, follows Cloud Run idioms, survives instance restarts. Con: Requires Cloud Scheduler infrastructure provisioning. Calendar sync runs serially within a single request — acceptable for single-tenant (bounded user count).
