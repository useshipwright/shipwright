# ADR-004: Subcollections for transcript segments and note versions

## Context

Meetings can have thousands of transcript segments and multiple note versions. Options: store as arrays within meeting document, separate top-level collections, or Firestore subcollections.


## Decision

Use subcollections: /meetings/{id}/segments/{segmentId} and /meetings/{id}/notes/{version}. Speakers also as subcollection /meetings/{id}/speakers/{speakerId}.


## Consequences

Pro: No document size limit concern (Firestore max 1MB), efficient pagination on segments, atomic meeting deletion via recursive delete, natural hierarchical query. Con: Cannot query segments across meetings in a single query (needed for search — solved by having searchTokens on the parent meeting document and the separate /embeddings collection).
