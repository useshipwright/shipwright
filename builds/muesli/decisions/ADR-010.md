# ADR-010: PRD Correction: CAP-13: Health

## Context

PRD validation found a mismatch between the PRD and domain documentation.

**PRD claims:** GET /health/ready — readiness probe. Checks Firestore and GCS connectivity.

**Documentation says:** Cloud Run docs confirm readiness probes are currently in Preview/Beta (requires run.googleapis.com/launch-stage: BETA annotation). Production use should consider whether this is acceptable.

**Source:** gcp-cloud-run — overview.md

## Decision

Follow the domain documentation behavior, not the PRD claim. The implementation should: Cloud Run docs confirm readiness probes are currently in Preview/Beta (requires run.googleapis.com/launch-stage: BETA annotation). Production use should consider whether this is acceptable.

## Consequences

- PRD section 'CAP-13: Health' is overridden
- Implementation follows verified API behavior
- Reduces risk of runtime errors from incorrect assumptions
