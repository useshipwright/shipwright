# ADR-007: Standard response envelope for all API responses

## Context

Q-009 asked about response format. Need consistency across 30+ endpoints for frontend consumption and error handling.


## Decision

All successful responses: { data: T, meta?: { cursor?, hasMore?, warning? } }. All error responses: { error: { code: number, message: string, details?: object } }. Fastify's setErrorHandler formats validation errors (Zod), auth errors, and unexpected errors into this envelope. HTTP status codes are canonical; the envelope provides structure.


## Consequences

Pro: Predictable client-side parsing, cursor-based pagination built into envelope, warnings for non-fatal issues (e.g., "notes were edited, regeneration will overwrite"). Con: Minor overhead wrapping simple responses.
