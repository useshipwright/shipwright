# ADR-008: Zod for request validation instead of Fastify's built-in JSON Schema

## Context

PRD requires Zod for input validation (REQ-032). Fastify natively uses Ajv/JSON Schema. Need to bridge the two.


## Decision

Use Zod schemas as the source of truth for request validation. Convert to JSON Schema via zod-to-json-schema for Fastify route schema registration (enables automatic 400 errors and OpenAPI generation). Zod schemas also used for TypeScript type inference. Validation runs at the Fastify schema level (Ajv) for performance; Zod schemas serve as the authoring format and type source.


## Consequences

Pro: Single schema definition for validation + types, better developer ergonomics than raw JSON Schema, composable schemas. Con: zod-to-json-schema conversion adds a build-time dependency. Edge cases where Zod features don't map cleanly to JSON Schema (e.g., refinements) must be handled in preHandler hooks instead.
