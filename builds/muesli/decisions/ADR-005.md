# ADR-005: Adapter pattern for all external dependencies

## Context

The system integrates with 7+ external services (Firestore, GCS, Pub/Sub, Deepgram, Whisper, Google STT, Claude, Vertex AI embeddings, Google Calendar). Testing with real services is slow and expensive.


## Decision

Every external dependency is accessed through a typed adapter interface. Adapters are injected via Fastify decorator pattern at startup. Tests mock at the adapter boundary. Adapter interfaces defined in src/adapters/*.ts, implementations in src/adapters/impl/*.ts.


## Consequences

Pro: Fast isolated tests, pluggable backends (transcription is explicitly pluggable), clear dependency boundaries, easy to add new implementations. Con: Slight indirection. Justified by the high number of external integrations.
