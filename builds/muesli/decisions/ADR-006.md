# ADR-006: Pub/Sub push subscriptions instead of pull for audio processing

## Context

Audio processing must be async. Options: Pub/Sub pull subscription with a long-running consumer, Pub/Sub push to Cloud Run endpoint, or Cloud Tasks.


## Decision

Use Pub/Sub push subscription delivering to /internal/process-audio on Cloud Run. Push subscriber authenticated via OIDC. Retry policy: 3 attempts, exponential backoff 10s-600s. Dead-letter topic for exhausted messages. Meeting marked failed on dead-letter.


## Consequences

Pro: No separate consumer process, scales with Cloud Run, automatic retry, dead-letter built in. Con: Processing timeout limited to Cloud Run request timeout (set to 3600s). Long meetings (3+ hrs) may need chunked processing — the pipeline already processes per-segment for streaming, so batch mode can reuse the same chunked approach.
