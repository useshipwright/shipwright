# ADR-011: PRD Correction: 7) Security Principles

## Context

PRD validation found a mismatch between the PRD and domain documentation.

**PRD claims:** Input validation via Zod on all request bodies and query params

**Documentation says:** Fastify uses Ajv v8 for JSON Schema validation by default, not Zod. While Zod can be used alongside Fastify (e.g., zod-to-json-schema), it's not the native validation approach. The PRD should clarify whether Zod replaces or supplements Fastify's built-in Ajv validation.

**Source:** fastify — overview.md

## Decision

Follow the domain documentation behavior, not the PRD claim. The implementation should: Fastify uses Ajv v8 for JSON Schema validation by default, not Zod. While Zod can be used alongside Fastify (e.g., zod-to-json-schema), it's not the native validation approach. The PRD should clarify whether Zod replaces or supplements Fastify's built-in Ajv validation.

## Consequences

- PRD section '7) Security Principles' is overridden
- Implementation follows verified API behavior
- Reduces risk of runtime errors from incorrect assumptions
