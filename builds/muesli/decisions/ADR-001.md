# ADR-001: Firestore native vector search instead of external vector DB

## Context

Q-001 flagged that Firestore doesn't natively support vector search. However, Firestore added vector search with HNSW indexes in 2024. The alternative was adding Pinecone, pgvector, or Vertex AI Matching Engine as a separate dependency.


## Decision

Use Firestore's native vector field type with HNSW index (cosine distance, 768 dimensions). Store embedding chunks in a top-level /embeddings collection with userId scoping. Use findNearest() for vector queries.


## Consequences

Pro: No additional infrastructure, consistent data model, automatic scaling, simpler deployment. Con: Less mature than dedicated vector DBs, potential performance ceiling at very large scale (millions of chunks). Acceptable for single-tenant self-hosted where corpus size is bounded. If performance degrades, can migrate to Vertex AI Vector Search later since the embedding format is portable.
