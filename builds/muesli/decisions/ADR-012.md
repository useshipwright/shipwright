# ADR-012: PRD Correction: 8) IAM Requirements — Runtime Service Account

## Context

PRD validation found a mismatch between the PRD and domain documentation.

**PRD claims:** roles/storage.objectAdmin (GCS)

**Documentation says:** GCS/IAM docs recommend roles/storage.objectCreator for upload-only or roles/storage.objectUser for upload+download+delete. roles/storage.objectAdmin provides full object control which is overprivileged if the service only needs upload, download, and delete. However, since Muesli needs to create, read, and delete audio files, roles/storage.objectUser would be more appropriate than objectAdmin.

**Source:** gcp-iam — overview.md, gcp-cloud-storage — overview.md

## Decision

Follow the domain documentation behavior, not the PRD claim. The implementation should: GCS/IAM docs recommend roles/storage.objectCreator for upload-only or roles/storage.objectUser for upload+download+delete. roles/storage.objectAdmin provides full object control which is overprivileged if the service only needs upload, download, and delete. However, since Muesli needs to create, read, and delete audio files, roles/storage.objectUser would be more appropriate than objectAdmin.

## Consequences

- PRD section '8) IAM Requirements — Runtime Service Account' is overridden
- Implementation follows verified API behavior
- Reduces risk of runtime errors from incorrect assumptions
